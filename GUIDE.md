# Руководство по работе со скиллами Plata

## Где всё лежит

```
/var/www/plata-marketplace/plugins/plata/skills/
├── plata-brief/SKILL.md       ← ТЗ для мастера
├── plata-price/SKILL.md       ← Расчёт стоимости
├── plata-offer/SKILL.md       ← КП клиенту
├── plata-reply/SKILL.md       ← Ответ на жалобу
├── plata-post/SKILL.md        ← Пост в соцсети
├── plata-collection/SKILL.md  ← Новая коллекция
├── plata-bizletter/SKILL.md   ← Деловое письмо
├── plata-decide/SKILL.md      ← Бизнес-решение
├── plata-debrief/SKILL.md     ← Итог дня
└── plata-n8n/SKILL.md         ← Автоматизация
```

---

## Как отредактировать скилл

1. Открой нужный `SKILL.md` в редакторе или через Claude
2. Измени инструкции или шаблоны внутри файла
3. Сохрани → изменения вступят в силу при следующем вызове скилла

**Через Claude:** просто напиши:
> "Отредактируй plata-reply — добавь отдельный паттерн для ситуации когда клиент хочет вернуть деньги"

---

## Как добавить новый скилл

1. Создай папку: `skills/plata-НАЗВАНИЕ/`
2. Создай файл `SKILL.md` внутри со структурой:

```markdown
---
name: plata-название
description: Одна строка — что делает скилл и когда его использовать.
argument-hint: [подсказка для аргументов]
---

Инструкции для Claude.
Описывай роль, формат вывода, правила.
```

3. Закоммить и запушить (см. ниже)

**Через Claude:** просто напиши:
> "Создай новый скилл plata-warranty — для написания гарантийных документов"

---

## Как обучить скилл (улучшить качество)

Скилл — это промпт. Качество зависит от точности инструкций.

**Если результат не устраивает:**
1. Запусти скилл и получи плохой результат
2. Скажи Claude: "В plata-reply не хватает..."
3. Claude исправит `SKILL.md` напрямую

**Что работает хорошо:**
- Конкретные правила вместо общих ("не длиннее 5 предложений" вместо "кратко")
- Примеры плохого и хорошего вывода прямо в скилле
- Явный список "чего не делать"
- Типизация ситуаций (если скилл для разных сценариев — опиши каждый)

**Шаблон секции "Обучение" в SKILL.md:**
```markdown
## Примеры

✅ Хороший ответ:
[пример]

❌ Плохой ответ:
[пример]
```

## Как сохранить изменения в GitHub

```bash
cd /var/www/plata-marketplace/plugins/plata

# Посмотреть что изменилось
git status

# Добавить изменения
git add skills/plata-reply/SKILL.md   # конкретный файл
# или все сразу:
git add .

# Закоммитить
git commit -m "plata-reply: добавлен паттерн для возврата"

# Отправить на GitHub
git push
```

---

## Как откатить скилл к предыдущей версии

```bash
# Посмотреть историю изменений
git log --oneline skills/plata-reply/SKILL.md

# Откатить конкретный файл к прошлому коммиту
git checkout <hash> skills/plata-reply/SKILL.md
git commit -m "plata-reply: откат к предыдущей версии"
git push
```

2. Зафиксировать что изменилось в commit message
3. Запушить в GitHub
